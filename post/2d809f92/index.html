compile pcl with vtk qt support from source on windows

发表于 2018-01-22 | 更新于: 2018-01-22 | 分类于 pcl
字数统计: 2,563 | 阅读时长 ≈ 16

Compile pcl with vtk qt5 support from source on windows 10.

Part 1: Guide

Tips
we’d better to use SSD to compile source code instead HDD. So put source code in C folder on windows.

prerequisit

software

  • vs: vs2015 installer
  • qt: qt-opensource-windows-x86-msvc2015_64-5.7.1.exe

  • pcl installer: https://github.com/PointCloudLibrary/pcl/releases/download/pcl-1.8.1/PCL-1.8.1-AllInOne-msvc2015-win64.exe

  • vtk source: https://github.com/Kitware/VTK/archive/v8.0.1.zip

  • pcl source: https://github.com/PointCloudLibrary/pcl/archive/pcl-1.8.1.zip

version

window: 10
qt: 5.7.1
qmake: 3.0
qtcreator: 4.2.0

cmake/gui: 3.10.0
vtk: 8.0.1 (source)
pcl: 1.8.1 (source)

$ qmake -v

Using Qt version 5.7.1 in C:/Qt/Qt5.7.1/5.7/msvc2015_64/lib

$ qtcreator -version

$ cmake -version
cmake version 3.10.0

install qt

install vs2015

install vs2015 (pass)

install qt

install qt path to C:\Qt\Qt5.7.1

add the following to system path.

C:\Qt\Qt5.7.1\5.7\msvc2015_64\bin
C:\Qt\Qt5.7.1\Tools\QtCreator\bin

install Qt VS Tools

start vs2015, and install Qt Visual Sutido Tools and restart vs2015.

Tools---> Extensions and Update... ---> online search qt ---> install Qt Visual Sutido Tools ---> OK

new menu Qt VS Tools will be added to VS Main menu.

configure qt for vs

follow steps

Qt VS Tools ---> Qt Options ---> Add name and path

Add qt path for vs2015 so that we can find %PATH%\bin\qmake.exe

Name: Qt 5.7.1
Path: C:\Qt\Qt5.7.1\5.7\msvc2015_64\

new qt project from vs

start vs2015, and new Qt Gui Application.

Compile and run first qt program.
Success.

install pcl

install pcl by installer

start pcl allInOne installer and install pcl to C:\Program Files\PCL 1.8.1\

choose to install OpenNI2 to C:\Program Files\PCL 1.8.1\3rdParty\OpenNI2\

this process will add 4 system env.

PCL_ROOT          C:\Program Files\PCL 1.8.1
OPENNI2_INCLUDE64 C:\Program Files\PCL 1.8.1\3rdParty\OpenNI2\Include\
OPENNI2_LIB64     C:\Program Files\PCL 1.8.1\3rdParty\OpenNI2\Lib\ 
OPENNI2_REDIST64  C:\Program Files\PCL 1.8.1\3rdParty\OpenNI2\Redist\

configure env

add env to Path.

%PCL_ROOT\bin
%PCL_ROOT%\3rdParty\FLANN\bin
%PCL_ROOT%\3rdParty\OpenNI2\Redist
%PCL_ROOT%\3rdParty\OpenNI2\Tools
%PCL_ROOT%\3rdParty\Qhull\bin
%PCL_ROOT%\3rdParty\VTK\bin

copy winpap to C:\Program Files\PCL 1.8.1\3rdParty\, and 3rdParty folder looks like this:

compile vtk

configure source

we need to compile vtk to replace C:\Program Files\PCL 1.8.1\3rdParty\VTK\

$ cmake-gui

and configure

source dir   C:/VTK-8.0.1
build dir    C:/VTK-8.0.1/build

configure generator to Visual Studio 14 2015 Win64

(1) With VTK = 8.0.1:

VTK_Group_Qt  ON

configure again, and then QT_QMAKE_EXECUTABLE and VTK_QT_VERSION will appear.

(2) we need to change VTK_QT_VERSION from 4 to 5.

VTK_QT_VERSION     5  # by default 4

(3) change VTK_RENDERING_BACKEND from OpenGL to OpenGL2.

VTK_RENDERING_BACKEND OpenGL2 # by default OpenGL

(4) change install path from C:\Program Files\VTK\ to D:\VTK\ so that we can have write privileges.

CMAKE_INSTALL_PREFIX  D:/VTK  # by default C:/Program Files/VTK

or

vim C:\VTK-8.0.1\build\cmake_install.cmake

#set(CMAKE_INSTALL_PREFIX "C:/Program Files/VTK")
set(CMAKE_INSTALL_PREFIX "D:/VTK")

otherwise, when we install, error will occur:

1>  -- Install configuration: "Release"
1>  CMake Error at cmake_install.cmake:36 (file):
1>    file cannot create directory: C:/Program Files/VTK/lib/cmake/vtk-8.0.
1>    Maybe need administrative privileges.

(5) we should also notice BUILD_SHARED_LIBS

BUILD_SHARED_LIBS  ON

(6) Configure and warning will occur

CMake Deprecation Warning at ThirdParty/libproj4/vtklibproj4/cmake/policies.cmake:2 (cmake_policy):
  The OLD behavior for policy CMP0022 will be removed from a future version
  of CMake.

  The cmake-policies(7) manual explains that the OLD behaviors of all
  policies are deprecated and that a policy should be set to OLD only under
  specific short-term circumstances.  Projects should be ported to the NEW
  behavior and not rely on setting a policy to OLD.
Call Stack (most recent call first):
  ThirdParty/libproj4/vtklibproj4/CMakeLists.txt:43 (include)

We can omit this warning.

If everything goes well, then generate VTK.sln

compile solution

Open VTK.sln in vs2015 and build with Release X64.

install vtk

Part 2: New Project

create pcl project with vtk and qt support.

Tips:

Use .props file to manage include path, library path and linker libraries.

set linker libraries

Tips: use bash to collect libraries instead of manually input many libraries.

eg. for VTK release version libs

cd D:\VTK\lib
ls *8.0.lib > names.txt

vim names.txt
# vim steps: (1)lower: CTLR+q (2)lower: 300j (3) upper: I (4); (5)esc

# process names to look like this:  
;QVTKWidgetPlugin.lib
;vtkalglib-8.0.lib
;vtkChartsCore-8.0.lib
;vtkCommonColor-8.0.lib
...
...

Edit VTK64.props for include, library, and linker libraries.

# edit VTK64.props
<IncludePath>
    C:\Program Files\PCL 1.8.1\include\pcl-1.8;
    ...
</IncludePath>

<LibraryPath>
    C:\Program Files\PCL 1.8.1\lib;
    ...
</LibraryPath>

<AdditionalDependencies>
    ...
    ...
    ;QVTKWidgetPlugin.lib
    ;vtkalglib-8.0.lib
    ;vtkChartsCore-8.0.lib
    ;vtkCommonColor-8.0.lib
    ...
    ...
</AdditionalDependencies>
